Задача 9 — C Format String Vulnerability
Идея: printf(argv[1]); вместо printf("%s", argv[1]);
Уязвимост: Нападателят подава форматиращи спецификатори (%x %x ...), което може да изчете стек или дори да доведе до RCE (в зависимост от контекста/платформа).
Експлойт: Стартирай компилираното приложение с аргумент %x %x %x и виж изход.
Поправка:
    • Винаги подавай format string: printf("%s", argv[1]);.
Приемни критерии:
    • В vulnerable извежда нещо и при payload от форматиращи спецификатори.
    • В fixed показва само входния низ; няма изтичане.
Чести капани:
    • На Windows липсва gcc — тестът да се skip-ва при липса (провери shutil.which('gcc')).
    • Относителни пътища към .c файла (решено чрез абсолютни пътища към repo root).
Hints:
    • Hint 1: “Защо printf(argv[1]) е опасно?”
    • Hint 2: “Какво правят %x или %n?”
    • Hint 3: “Как да направиш кода устойчив на промяна на формата от входа?”
Разширение: Добави -fstack-protector, ASLR, компилационни флагове; демонстрирай Address Sanitizer (ако е наличен).
