Задача 1 — SQL Injection (автентикация)
Идея: Входните стойности за потребител/парола се вкарват директно в SQL заявка с конкатенация на низове.
Уязвимост: SELECT ... WHERE username='%s' AND password='%s'.
Експлойт: Потребителското име включва инжекция, която “коментира” паролата.
    • Пример payload (работещ):
username: ' OR '1'='1' -- 
password: x
(Забележка: интервал след -- е важен в много СУБД.)
    • Очакван ефект: Логин без познаване на парола.
Поправка: Параметризирани заявки/placeholder-и:
SELECT id FROM users WHERE username=? AND password=? (sqlite), съответно cursor.execute(sql, (u,p)).
Приемни критерии (tests):
    • Функционален: валиден user/pass → успех.
    • Уязвимост: payload-ът по-горе → успех (в vulnerable).
    • Фикс: payload-ът → неуспех (в fixed).
Чести капани:
    • Използване на format()/f-strings вместо параметризация.
    • Избягване на -- коментар → AND има по-висок приоритет от OR → експлойтът не работи.
Hints (подсещащи):
    • Hint 1: “Защо "' OR '1'='1" не винаги заобикаля паролата? Помисли за приоритета на AND/OR.”
    • Hint 2: “Как да накараш заявката да игнорира проверката за парола? Помисли за SQL коментар.”
    • Hint 3: “Коя е разликата между '? и конкатенация на низове в sqlite3?”
Разширение: Логване на грешки без да се издава SQL; използване на ограничени привилегии в БД.
